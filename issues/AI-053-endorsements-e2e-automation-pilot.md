## Description

Endorsements identified as the strongest candidate for an isolated end-to-end workflow pilot: well-documented process, straightforward, good precedent in codebase and version history.

The goal: take a linear ticket → agent produces subtask breakdown → agent produces draft PR(s) → engineer reviews as if reviewing a mid-level hire's output.

This is the concrete demonstration of the direction of travel agreed in Dev AI Practices session (2026-05-18). Success here creates the playbook for scaling to other workflows.

## Acceptance criteria

- [ ] Scope the endorsement workflow end-to-end — identify all repos and steps involved
- [ ] Set up the linear → subtask breakdown step (immediate value, usable without full automation)
- [ ] Extend to linear → draft PR for the endorsement process
- [ ] Engineer review confirms: "I'd be comfortable stamping my name on this"
- [ ] Document what worked and what needed repointing — becomes the playbook
