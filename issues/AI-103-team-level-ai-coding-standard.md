The team lacks a shared standard for AI-assisted code. This creates drift: some engineers use AI heavily, others don't, and there's no shared vocabulary for what "good" looks like.

## Action

- Define what a team-level AI coding standard covers: when to use AI, how to review AI-generated code, how to handle hallucinations, what to commit vs. discard
- Draft a starter version in collaboration with Chris and the engineering group therapy participants
- Encode architectural rules as linter checks where possible (see AI-104)

## Context

Different from a style guide — this is about the workflow and judgment layer, not syntax. The standard should be minimal enough to actually be followed.
