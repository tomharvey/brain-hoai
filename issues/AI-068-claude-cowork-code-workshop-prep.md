## Description

90-minute company-wide workshop on when and how to use Claude, Cowork, and Claude Code. Booked: **Tue 16 June, 11:30, in-person + remote**.

Core mental model:
- **Claude** — thinking partner. Conversational, exploratory, one-off tasks.
- **Cowork** — virtual employee. Scheduled autonomous agents, runs without you.
- **Claude Code** — building partner. File-aware, long-running, lives in the terminal (signposted as next step only, not demoed).

Key insight to land: friction comes from routing work to the wrong tool, not from the models themselves.

**Three new framing additions (fold into framing block and activity):**

1. **Intent first** (Fergus, 2026-06-11): the two failure modes — over-engineering and fire-and-forget — both come from reaching for the tool before the intent is clear. "What am I trying to produce and why?" is the question that comes before "which tool do I use?" Land this as a mindset slide before the routing model.

2. **Incremental building**: don't jump to hi-fi. Version zero should be a conversation, not a dashboard. Start small, get clarity, add fidelity. "When I see so many widgets in version zero, 95% of that is bullshit."

3. **Context over prompting**: the skill isn't writing better prompts — it's building better context. A well-loaded project does more than a perfectly crafted prompt. Most people try to write a better prompt when they get a bad answer; the real fix is better context. This reframes what co:work *is*: not a smarter prompt box, but the place where you build the context layer that makes the AI permanently more useful.
