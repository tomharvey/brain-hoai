## Key themes

### Flock O'Clock automation design

- Phoebe currently: manually copies template deck, titles it, saves it, then chases all contributors via Slack
- Goal: automate heavy lifting, unify slide formatting, reduce contributor friction
- Secondary goal: nudge company toward Claude adoption via a familiar, recurring task
- Run sheet spreadsheet already improved with Claude's help but needs cleanup before it can reliably drive automation

### Proposed architecture

- Skill holds the template and formatting logic, not the data (analogy: mail merge)
- Skill reads from a two-column spreadsheet: contributor name + their content
- Contributors feed content via a dedicated Slack channel (Flock O'Clock Planning)
  - Claude (now taggable in Slack via Anthropic's latest release) pings each contributor in their own thread
  - Contributors reply with bullet points or a link to their slides
  - Claude validates quality against Ed's comms playbook, gives feedback without Phoebe being the bad guy
  - Claude writes approved content back into column B of the spreadsheet
- Deck rebuilds automatically each time a contributor submits
- Phoebe gets a live view and can catch anyone wildly off-track in real time
- Intermediate step: Claude generates a slide-by-slide outline doc first for Phoebe to verify order and segment breaks before building the full deck

### Flexibility for contributors

- Text-only updates: bullet points into Slack thread, Claude formats and brands them
- Multimedia (charts, tables, images): contributor builds their own slides and drops in — Claude can't manipulate non-text
- Contributors who want full control (e.g. Adam Smith for distribution) can still drop finished slides directly into Google Slides
