# Alex Dyball — AI Discovery

Discovery call with [[alex-dyball]] as part of the Head of AI round of 1:1s. Alex is a pragmatic, results-oriented user — not interested in how it works, just what it can do. Already a solid adopter across three tools.

## Attendees

- [[alex-dyball]]
- Tom (me)

---

## Current AI usage

Alex uses three tools: **Granola**, **Claude**, and **ChatGPT**. Leans heavily on ChatGPT for speed despite acknowledging Claude is "better".

### Granola
- Transcription and note-taking on calls (not in-person — feels invasive, plus WiFi not always available)
- Action extraction from meetings
- Distribution team has a shared Granola folder so team members can search across each other's calls
- Uses Granola search directly for cross-call queries (e.g. "which brokers mentioned taxi fleets?")

### Claude
- **Presentations**: recently adopted Mima's company presentation skill (~2 days ago)
- **Data analysis via Looker**: Kirsty helped connect Looker MCP yesterday. Previously was downloading docs from Looker and processing manually — same outcome, much faster now

### ChatGPT (highest volume)
Four main use cases:
1. **Rough notes → broker comms**: Takes handwritten/pen-and-paper notes from face-to-face broker meetings, processes into follow-up emails
2. **Action extraction**: e.g. 17 tasks from an Anton meeting — "just tell me what I need to do"
3. **Broker account strategy**: Uses as a sounding board instead of pestering [[adam-smith]]. Helps contextualise strategy, word sensitive emails, profile account approaches
4. **Meeting performance review**: Scores his own meeting performance (and sometimes the broker's). Scores align with his gut feel. Has surfaced specific skill development areas — e.g. not pushing brokers for deadlines when they say "I'll send that over"

### Why ChatGPT over Claude
- Speed: "I just want it done in 30 seconds rather than four minutes"
- Familiarity: heavy personal use on phone, feels like "a friend"
- Tone: ChatGPT is "kinder", Claude is "more brutal"
- I suggested trying Haiku for speed — he wasn't aware of model switching

---

## Opportunities identified

### Renewal nudge automation
- Wants: auto-nudge brokers 30 days before quote renewal dates
- **Blocker**: political nuance — multiple brokers may have quoted, one won, approaching the loser is awkward
- Data quality issue: underwriters don't record detailed loss reasons ("sulky note and move on")
- Billy's human memory still needed for relationship context that isn't in any system
- Need better data capture at loss stage before this can be automated

### Granola → HubSpot integration
- Currently distribution team shares a Granola folder for continuity (if someone leaves)
- Acknowledged value in pushing Granola summaries into HubSpot but not doing it yet
- Underwriters already have Claude → HubSpot integration for file parsing and task management

### Workshop attendance
- Has missed AI workshops due to broker meeting conflicts
- Sophie has the same problem — consistently can't make AI Breakfast on Fridays
- Distribution/sales teams will always prioritise client-facing time over internal sessions

---

## Assessment

Alex is a confident, self-sufficient AI user who doesn't need hand-holding. His usage pattern is:
- **High volume, low complexity** — lots of quick ChatGPT interactions for comms, strategy, and self-review
- **Pragmatic** — doesn't care about how LLMs work, just results
- **No burning unmet need** — happy to adopt things as they become available

Key observation: the meeting performance scoring use case is creative and genuinely useful for self-development. Worth sharing as a pattern with others in the enablement programme.

The main gap is the renewal nudge workflow, but that's blocked by data quality in underwriting, not by AI capability.
