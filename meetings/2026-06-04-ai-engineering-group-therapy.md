## Key themes

### Brain rot — real but managed
- **Rob**: speed gains are real but syntax knowledge atrophying; mitigated by embedding Flock coding standards + functional programming principles into Claude Code; runs Codex adversarial review as a final pass
- **Javier**: "taking the craft out of the work — becomes associate factory." Lost: deep debugging satisfaction. The learning comes from getting your hands dirty, which AI reduces
- **Ismael**: job scope expanded (can tackle frontend now); satisfaction from solving hard bugs independently is gone; but likes the job more overall — "my scope has definitely expanded"
- All three lived pre-AI development; hope that instilled instincts pay off vs. a career started AI-first (Rob's position)

### Speed and implicit expectations
- **Javier**: runs Claude at 5pm, checks back at 8pm — work bleeding into evenings without explicit pressure; high output expectations implicit, not stated
- **[[geran]]**'s view (product perspective, reported by Rob): no tangible speed increase in project completion in the AI era — engineers going faster but project throughput the same from PM's vantage point
- Rob: biggest blocker is "finding the valuable work" not writing the code — could go 2–3x faster with more tickets; suggests some teams moving to 2 PMs per engineer ratio

### Paradigm shifts — generational comparison
- Tom: MVC frameworks / Rails
- Ismael: AWS / cloud was his paradigm shift
- Javier: entered during low-code/no-code collapse, spent early career fighting dodgy stakeholder investments in those tools
- Parallel drawn to low-code/no-code hype: "this means we won't need expensive engineers" — same claim heard before, never fully materialised

### Dopamine and craft
- Lost hit: days lost to deep debugging, "all-nighters lost to code"
- Gained hit: "all-powerful ego" from running parallel agent threads; Javier compared to doom-scrolling
- Tom's shift: now conscious about what he does/doesn't use AI for; gets the hit from learning to use AI well and solving agentic engineering problems; no longer uses AI for English language messaging

### Adversarial review pattern (Rob)
- Rob's current gold standard: Grill me skill (forces Claude to exhaust decision trees and verify assumptions before acting) → generate code with Flock standards embedded → Codex adversarial review pass
- Comparing this to being an engineering manager: you're picking among smart people's opinions, not writing yourself — reframe to avoid brain rot feeling

### Job security
- Ismael: not worried — "once software engineering is fully automatable end to end, every single white collar role is automatable end to end"
- Key unsolved problems: jagged intelligence (answer quality varies with phrasing) + context window / persistent memory
- Rob: less worried about job security than about quality — "how do I know what it's given me is right?"
- Alex Dyball's question (from prior session, referenced): "code review is dead now — if I'm in a thing I don't understand, how do I validate its output?" → answered with: require it to explain, not just change its mind; ask for receipts
