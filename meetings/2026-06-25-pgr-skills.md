## Key themes

### Background and inspiration

- Francesco and Eraaz built a PGR/1:1 tracking tool ("Pace") as inspiration
  - Over-engineered: 15–20 features, too complex to roll out broadly
  - Heavy reliance on existing Granola transcripts and Notion content as input
- Goal: distill to 1–2 core features relevant to PGRs and ongoing performance management

### What the team wants to build

- Two core use cases:
  - Help managers prepare for PGRs and 1:1s
  - Help team members prepare their own PGRs and 1:1s
- Key value: autonomous tracking across months so nothing gets forgotten at review time
  - Eraaz example: 4 months of talent work with Rakhee would be hard to summarise accurately without it
- Lightweight co:work setup envisioned:
  - Folder for 1:1 transcripts
  - File tracking goals and completed work
  - Scheduled prompts (e.g. Monday action items, Friday reflection)

### Concerns and constraints

- AI adoption gap: many staff (underwriters, finance) not yet at a base level of AI fluency
  - Tool only works well if people are actually using it to capture 1:1s
  - Input quality matters: transcribed sessions yield far better results than manual summaries
- Over-reliance risk: Rakhee's concern that people stop adding their own judgment
- Pace is too Francesco-specific to hand to others directly; starting from first principles is better

### Phased approach

- Current PGRs (launching next week, running 2.5–3 weeks): timeline too tight for a tool
  - This cycle treated as starting point: capture it, track forward to the next PGR
- Near-term: build a simple, copyable prompt that creates a co:work skill/project
  - Optional but with explicit opt-in commitment — not passive "do it or don't"
  - People can customise the prompt to fit their role and how they run 1:1s
- Longer term: PGR cycle moving to quarterly (next year); more robust co:work skill to follow
  - Bob integration being worked on, which would help with data connectors
