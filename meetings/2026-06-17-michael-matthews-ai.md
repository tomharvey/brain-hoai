## Key themes

### Current AI usage — Stage 1-2 assessment

Michael's usage is ad hoc and not yet habitual:
- **Presentation skill** — most frequent use. Workflow: dumps slide content/commentary into Claude, gets polished output. Has progressed from "make existing slides nicer" to "generate slides from data."
- **Excel plugin** — highlights cells in claims spreadsheets, runs trend analysis in background. Handy for quick analysis alongside manual work.
- **Habit gap** — defaults to Snowflake/Looker directly rather than reaching for Claude first. Doesn't think to use AI for ad hoc tasks because they don't feel repeatable.
- **One-shot tendency** — types long, comprehensive prompts rather than having iterative conversations. "I probably over-type it."

**Assessment: Stage 2 (low-medium confidence).** Has context awareness (Excel plugin, project setup) and uses tools, but not yet iterating conversationally. Defaults to old workflows. The habit gap is the primary blocker, not capability or mindset.

### Large cases import project — stalled but promising

Michael started building a large cases import tool — partially complete, stalled due to workload:
- Mapped claim listing columns from 10-15 insurers into a standardized Excel template
- Goal: drop in any claims listing, auto-aggregate into one sheet
- Further ambitions: auto-calculate driver age from DOB, embed CC importer, build as a skill within the Excel plugin

**Large deals brain** — Michael's next ambition: upload all historical large cases with qualitative notes (broker, segment, excess, outcomes) to build a searchable repository. Sees value in pattern matching across cases that recur annually.

Large cases files are scattered across his personal Drive, Milan's folders, and Will's (predecessor) folders.

### Mindset and blockers

- **"Too ad hoc to automate"** — Michael's framing. Reframe from Tom: even one-off tasks benefit from offloading the clicking-around so thinking time stays on strategy.
- **Time-poor as advantage** — can't afford to over-engineer. Contrast with Matt Lees and [[jake-wood]] who went heads-down and over-built. Michael should do 4-minute increments, not blocked-out days.
- **Wants fundamentals** — interested in how models work, not just possibilities. Risk: courses go out of date fast.
- **Discussed pricing/actuarial skill with Milan** — talked about it but never built it. Verdict: stop planning, just pick up the tool.

### Session outcome — created a project live

Tom and Michael created a Claude co:work project pointed at Michael's large cases folder during the call. Set up with Opus model. Next steps are small daily interactions, not a big build.

Discussed linking Whisper to Claude for voice input — Michael prefers talking to typing. Granola transcript of this meeting could itself be useful input.
