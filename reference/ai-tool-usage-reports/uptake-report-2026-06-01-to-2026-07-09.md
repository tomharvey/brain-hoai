## Method (short form)

Raw request counts are never compared across products. Each person gets: **within-product percentile** (vs other users of the same product), **product class** (chat < assisted < agentic), **breadth** (products used), and **sustained** (active in both June and July). Buckets:

| Bucket | Criteria |
|---|---|
| **power** | ≥60th percentile in an agentic-class product (Cowork, Claude Code, Cursor, Office Agents, Research) + 3+ products + active both months |
| **regular** | 2+ products, or ≥75th percentile in one |
| **light** | everything below |
| **dormant** | scored person with zero telemetry (none this period) |

Buckets are a **screen, not a score** — the Stage 1–4 capability assessments remain ground truth; the telemetry flags where to look.
